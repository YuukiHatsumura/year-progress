## 1. アプリの目的

相手の発言テキストから以下を推定し、**相手に最適化された会話スタイルで応答するAI**。

| 推定対象 | 説明 |
|--------|------|
| 感情 | Valence / Arousal ベクトル |
| 会話文脈 | 前後の流れ・状態遷移 |
| 社会距離 | 親しさ・フォーマリティ |

**対象機能：会話スタイルの適応**

### 応答比較例

| | 発言例 |
|--|--------|
| 相手 | 「今日は仕事きつかった」 |
| ❌ 冷たい応答 | 「そうなんだ」 |
| ✅ 適応型応答 | 「仕事きつかったんだね。お疲れさま。」 |

> **重要ポイント**: 音声トーンではなく **言語表現の適応** が本設計の対象。

## 2. システム全体構造

### 基本パイプライン

```
text input
  ↓
tokenize
  ↓
sentiment detection
  ↓
modifier detection         ← 修飾語「めっちゃ」「結構」など
  ↓
negation detection         ← 「嫌いじゃない」「悪くない」など
  ↓
context shift detection    ← 「でも」「けど」などの接続詞
  ↓
emotion vector             ← (Valence, Arousal)
  ↓
conversation state update
  ↓
style selection
  ↓
response generation
```

### 拡張版パイプライン

```
user text
  ↓
NLP parsing
  ↓
emotion inference          ← 感情ベクトル推定
  ↓
intent detection           ← 意図分類（共感希求 / 相談 / 愚痴 / 情報共有）
  ↓
conversation state update  ← 状態差分更新
  ↓
personality estimation     ← 論理型 / 感情型 / 行動型
  ↓
style decision             ← tone = f(emotion, distance, personality)
  ↓
response planning          ← acknowledge → mirror → suggestion
  ↓
response generation        ← テンプレ or LLM
```

## 3. 感情推定モデル

感情はカテゴリ分類よりも **連続値ベクトル** のほうが自然に表現できる。

### Valence-Arousal モデル

| 軸 | 意味 |
|----|------|
| Valence | 快（+1.0）〜 不快（-1.0） |
| Arousal | 興奮（+1.0）〜 落ち着き（-1.0） |

### 感情座標例

| 感情 | Valence | Arousal |
|------|---------|---------|
| 嬉しい | +0.8 | +0.7 |
| 悲しい | -0.7 | -0.2 |
| 怒り | -0.8 | +0.9 |
| 不安 | -0.5 | +0.5 |
| 安心 | +0.6 | -0.4 |

### ベクトルが制御するパラメータ

```
emotion_vector (V, A)
  ├── 共感強度
  ├── 応答長さ
  ├── 柔らかさ
  └── 励まし度
```

## 4. 日本語感情推定の重要知見

### ① 日本語は感情を直接言わない

日本語では感情を曖昧表現に包む傾向がある。

| 発言 | 実際の感情 |
|------|-----------|
| 「まあいいか」 | 諦め / 微ネガ |
| 「仕方ない」 | 諦め / 不満 |
| 「別にいい」 | 拒絶 / 強ネガ |

---

### ② 文末が感情シグナル

文末助詞・語尾が感情の重要な手がかりになる。

```
「いいと思う」       → 普通のポジ
「いいと思うけど」   → 逆接あり → ネガ寄り
「いいと思うけどね」 → 含み強い → さらにネガ寄り
```

**重み配分（推奨）**

```
本文 : 文末 = 70% : 30%
```

---

### ③ 接続詞は感情反転シグナル

以下の接続詞が出現すると感情が反転 or 弱まる。

| 接続詞 | 効果 |
|--------|------|
| でも | 反転（ポジ→ネガ） |
| けど | 反転（ポジ→ネガ） |
| ただ | 留保（強度低下） |
| まあ | 曖昧化（中立寄り） |

---

### ④ 否定形は「弱ポジ」として扱う

| 発言 | 解釈 |
|------|------|
| 「嫌いじゃない」 | 弱ポジ（好意あり） |
| 「悪くない」 | 弱ポジ（肯定） |
| 「ダメではない」 | 条件付き肯定 |

**否定形 → 弱ポジ変換** をルールとして実装する。

---

### ⑤ 修飾語で感情強度が変わる

```
いい             → Valence = +0.4
結構いい         → Valence = +0.6
めっちゃいい     → Valence = +0.8
最高             → Valence = +1.0
```

**修飾語強度テーブル（例）**

| 修飾語 | 強度係数 |
|--------|---------|
| ちょっと | 0.5 |
| 結構 | 1.3 |
| かなり | 1.5 |
| めっちゃ / すごく | 1.8 |
| 最高 / 最悪 | 2.0 |

## 5. 感情は差分で発生する

### 数式

```
Emotion = Outcome − Expectation
```

### 会話における更新式

```
emotion(t) = emotion(t-1) + Δemotion
```

### 例

```
A: 「この企画どう？」
B: 「まあいいと思う」
```

- 単語単体: ポジ寄り
- 文脈評価: 「まあ」が期待を下げている → **ややネガ**

> 前ターンの期待値と今ターンの評価の **差分** で感情を動かす設計が重要。

```python
conversation_state = {

    "last_intent":    str,     # 直前の発話意図

```

### 更新式

```
state_t = state_t-1 + Δstate
```

### 状態遷移の重要性

- 単発発話の感情だけでなく **会話の流れ（トレンド）** を追う
- 例: trust_level が高まるほど柔らかい敬語から砕けた表現に移行できる

## 9. 日本語の柔らかさ制御

### 柔らかさスペクトル

```
【直接】 違う
   ↓
【弱め】 違うと思う
   ↓
【やや柔】 少し違う気がする
   ↓
【最柔】 そういう見方もあるかも
```

### 柔らかさ調整語リスト

| 調整語 | 効果 |
|--------|------|
| と思う | 主観化 |
| かもしれない | 不確実性付与 |
| 気がする | 曖昧化 |
| 少し / ちょっと | 強度低下 |
| ～な感じ | やわらかい印象 |

### 実装イメージ

```python
def soften(text: str, directness: float) -> str:
    """
    directness: 0.0（最柔） ～ 1.0（直接）
    """
    if directness < 0.3:
        return add_softener(text, level="max")
    elif directness < 0.6:
        return add_softener(text, level="mid")
    else:
        return text
```

## 10. 共感を強くする技術

### ミラーリング

相手の言葉を反復することで共感を表現する。

```
相手: 「仕事きつかった」
AI  : 「仕事きつかったんだね。」
```

### 実装構造

```
keyword extract（重要語抽出）
  ↓
repeat phrase（繰り返し構文に変換）
  ↓
append empathy（お疲れさま / そうなんだ など追加）
```

### ミラーリング強度と感情の関係

| emotion_level | ミラーリング強度 |
|--------------|----------------|
| 高ネガ（-0.8以下） | 強（完全反復） |
| 中ネガ（-0.4 ～ -0.8） | 中（部分反復） |
| 中立〜ポジ | 弱（言い換え） |

## 11. 応答長さ制御

感情強度によって応答長を自動調整する。

| 感情強度 | 応答スタイル | 例 |
|---------|------------|-----|
| 高（強い感情） | 短文・直接的 | 「それは辛いね。」 |
| 中 | 通常長 | 「仕事きつかったんだね。お疲れ様。無理しないで。」 |
| 低（中立） | 長文可 | 詳細な説明・情報提供も可 |

### 実装イメージ

```python
def decide_response_length(arousal: float, valence: float) -> str:
    intensity = abs(valence) + abs(arousal)
    if intensity > 1.4:
        return "short"    # 3語〜10語
    elif intensity > 0.8:
        return "medium"   # 10語〜30語
    else:
        return "long"     # 30語以上も可
```

## 12. 優しい会話の条件

### 重要3要素

```
1. 共感（Empathy）   ← 相手の感情を受け止める
2. 理解（Understanding） ← 状況を把握していることを示す
3. 非否定（Non-denial）  ← 相手の感情を否定しない
```

### 優しい応答の構造

```
acknowledge  ← 「そうなんだね」「それは大変だったね」
  ↓
mirror       ← 相手のキーワードを繰り返す
  ↓
suggestion   ← （必要なら）次のアクション or 問いかけ
```

## 14. 応答生成方法

### 初期実装：テンプレート方式

感情ベクトルをテンプレートにマッピング。

```python
templates = {
    "sad":   "それは大変でしたね。",
    "angry": "それは納得いかないですよね。",
    "happy": "それは良かったですね！",
    "anxious": "心配になりますよね。",
}
```

### テンプレート + キーワード挿入

```python
def generate(emotion: str, keywords: list[str]) -> str:
    base = templates[emotion]
    mirror = f"{keywords[0]}んだね。" if keywords else ""
    return mirror + base
```

**例**:
- emotion = "sad", keyword = "仕事きつかった"
- → 「仕事きつかったんだね。それは大変でしたね。」

## 15. 性格タイプ推定

**「優しい」の定義は人によって異なる**。

### タイプ分類

| タイプ | 特徴 | 好む応答 |
|--------|------|---------|
| 論理型 | 原因・理由を重視 | 結論や解決策 |
| 感情型 | 共感・受容を重視 | 共感・傾聴 |
| 行動型 | 次のアクションを重視 | 具体的な提案 |

### 推定方法

- 発話パターンの蓄積から統計的に分類
- 「なんで？」「どうすれば？」→ 論理型 / 行動型
- 「つらい」「しんどい」 → 感情型

## 16. 会話意図（Intent）検出

> **研究的に最も重要なレイヤー**

### なぜ重要か

同じ発言でも意図が違えば最適な応答が変わる。

```
「仕事きつかった」
  ├── 共感が欲しい   → ミラーリング + 共感
  ├── 相談したい     → 問いかけ + 解決提案
  ├── 愚痴を言いたい → 受け止め + 同調
  └── ただの情報共有 → 軽い相槌
```

### 意図分類の重要性

**意図を誤ると AIが嫌われる**

| 誤り | 結果 |
|------|------|
| 共感希求 → アドバイス開始 | 「話を聞いてくれない」 |
| 愚痴 → 解決策提案 | 「そういうことじゃない」 |
| 情報共有 → 深刻な共感 | 「大げさ」 |

### Intent 分類ラベル（案）

```python
IntentType = Literal[
    "empathy_seeking",    # 共感希求
    "consultation",       # 相談
    "venting",            # 愚痴
    "information_share",  # 情報共有
    "question",           # 質問
    "action_request",     # 依頼
]
```

### Intent → 応答戦略マッピング

| Intent | 推奨応答戦略 |
|--------|------------|
| empathy_seeking | ミラーリング → 共感 |
| consultation | 受け止め → 問いかけ → 提案 |
| venting | 受け止め → 同調 → 共感 |
| information_share | 相槌 → 軽い反応 |
| question | 明確化 → 情報提供 |
| action_request | 確認 → 実行 or 代替案 |

| Intent 分類精度 | F1スコア |
| 応答適切性 | 人手評価（1-5点スケール） |
| 共感度評価 | 「気持ちを理解された」かのユーザー評価 |

## 20. 不足している重要レイヤー（追加）

### ① フィードバック学習ループ

現状の設計は **ワンショット推定** になっている。

```
現状:  入力 → 推定 → 応答
理想:  入力 → 推定 → 応答 → ユーザー反応 → 状態補正
```

**追加すべき要素**:
- ユーザーが応答を無視した → engagement 低下 → スタイル変更
- ユーザーが長文で返答した → trust_level 上昇

---

### ② マルチターン文脈管理

単一発話の感情推定だけでなく、**複数ターンの流れ** を管理する必要がある。

```python
class ConversationHistory:
    turns: list[Turn]  # 過去 N ターン保持
    summary: str       # 長期要約（LLM で生成）
    emotion_trend: list[EmotionVector]  # 感情推移グラフ
```

---

### ③ 曖昧性解決（Disambiguation）

日本語は主語省略・文脈依存が多いため、曖昧な発話を解決するレイヤーが必要。

```
「きつい」       → 肉体的 / 精神的 / スケジュール的？
「やばい」       → ポジ / ネガ（若者言葉では両方）
「ちょっといい？」 → 確認依頼 / 否定婉曲？
```

**対処**: 文脈 + 前ターン + Intent から補完推定

---

### ④ 倫理・安全フィルター

感情が極端に強い場合（深刻なネガ）への対処。

```
stress_level > 0.9 かつ valence < -0.8
  → 「専門家への相談」を示唆するレイヤーを挟む
```

---

### ⑤ パーソナライズ記憶

会話を重ねるほど精度が上がる仕組み。

```python
user_profile = {
    "preferred_intent_response": dict,  # 過去の反応から学習
    "personality_type": str,
    "social_distance_preference": float,
    "common_keywords": list[str],        # よく使う言葉
}
```
